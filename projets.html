Fiche descriptive : Dynamisation du site de l'activité (AP2)

Titre de la situation Dynamisation du site de l'activité de l'AP1 — base de données MySQL, accès PDO, authentification, programmation objet et RGPD
Descriptif Prolongement du site vitrine de l'AP1 vers une solution dynamique. Conception d'une base de données MySQL (modélisée avec Looping), implémentation d'un CRUD complet via PHP/PDO, authentification sécurisée par sessions, programmation orientée objet (panier virtuel de réservation d'évènements) et mise en conformité RGPD avec consentement de cookies. Ouverture des back-offices côté administrateur et front-office côté adhérent.
Besoins fonctionnels
  • Gestion des adhérents (inscription, photo, catégorie d'âge, identifiant + mot de passe haché)
  • Gestion des tarifs et des réductions par catégorie
  • Gestion du calendrier des séances/entraînements (date, plage horaire, objectif, présence)
  • Gestion des évènements (compétitions, tournois, représentations) avec résultats
  • Réservation de places par formulaire (adhérents et non-adhérents)
  • Gestion des données du formulaire de contact de l'AP1
Phases de travail
  1. MCD + base MySQL avec Looping, jeu d'essai sur Laragon
  2. Environnement Laragon et copie du site AP1
  3. Accès PDO (FrontOffice : insertion adhérents avec hashage + page profil ; BackOffice : listage par catégorie via combo box)
  4. Statistiques PDO : nombre/taux d'adhésions par catégorie, montant total annuel
  5. CRUD complet sur la rubrique Adhérent (Create, Read, Update, Delete)
  6. Authentification par sessions (formulaire, session_start, $_SESSION, déconnexion)
  7. Panier virtuel en PHP objet : classe Reservation + tableau d'objets (TableauDeReservationPossible, Panier) + table SQL Panier
  8. RGPD : consentement de cookies (codexworld / Cookiebot)
  9. Intégration globale : validation W3C, tests, mise en production sur SRVSIOC, mise à jour du portfolio
Architecture applicative Classes métier + Configuration.cfg.php + classe PDO (extensions MySQL) → connexion + ordres SQL + résultats. Composants : formulaires de modification, pages action, listes déroulantes, tableaux HTML, modules de recherche et d'authentification.
Période de réalisation Mars – Mai 2025 (cours/TP du 11/03 au 13/05/2025, évaluation finale le 13/05/2025)
Outils & technologies Backend : PHP 8 (PDO, sessions, POO, héritage, abstraction, MVC)
SGBD : MySQL via Laragon, phpMyAdmin
Modélisation : Looping (MCD)
Frontend : HTML5, CSS3, JavaScript, AJAX
Sécurité : password_hash(), $_SESSION, JWT
RGPD : codexworld / Cookiebot pour le consentement
IDE : Visual Studio Code
Validation : Validateur W3C
Production : serveur SRVSIOC du lycée
Lieu Lycée CCI de Nîmes
Cadre de réalisation Atelier de Professionnalisation 2 (AP2) — BTS SIO 1ère année option SLAM (groupe SIO1B)

Fiche descriptive : AP3 — Maison de la Culture (SQL Server & CodeIgniter)

Titre de la situation Analyse et développement d'un site web client/serveur avec SQL Server et Framework CodeIgniter — contexte Maison de la Culture
Descriptif Prolongement de l'AP2 : enrichissement de la base de données, migration MySQL → SQL Server avec SSMA, automatisation des règles de gestion par déclencheurs et procédures stockées en Transact-SQL, et reconstruction du site web côté client en MVC avec le Framework CodeIgniter. Authentification multi-rôles (invité, joueur, entraîneur, membre de l'organisation) avec redirection vers des vues spécifiques.
Domaines fonctionnels
  • Adhérents : héritage entre adhérent / entraîneur (spécialité, diplôme, expérience) / joueur (date d'entrée, renouvellement)
  • Évènements : tournois/compétitions/séances animés par un coach (avec remplaçant via réflexivité), inscriptions, présence et travail réalisé, lieu
  • Réunions internes : convocation des entraîneurs, ordre du jour ordonné, dates et objets
Travaux base de données
  1. Compléter le MCD avec Looping (héritage, réflexivité, contraintes de partition/exclusion/totalité)
  2. Migrer la base MySQL vers SQL Server avec SSMA
  3. Compléter la base SQL Server, créer un jeu d'essai
  4. Sauvegarder la BDD via génération de scripts SQL (SSMS)
Programmation Transact-SQL Procédures stockées :
  • Liste des adhérents ayant renouvelé leur adhésion
  • Liste des points à traiter (ordre du jour) d'une réunion
  • Liste des évènements suivis par un adhérent (présence + travail)
  • Nombre d'évènements suivis par un joueur entre 2 dates
  • Nombre d'heures d'entraînement suivies par chaque joueur
Déclencheurs (triggers) :
  • Génération automatique de login + mot de passe haché (HASHBYTES SHA2_512) à l'ajout d'un adhérent
  • Comptage automatique du nombre d'évènements par adhérent
  • Historisation dans HISTOADH en cas de non-renouvellement
Vues :
  • Évènements avec personnes présentes
  • Adhérents entraîneurs / Adhérents joueurs
Site web — CodeIgniter (MVC) Architecture du projet : 
application/
├── controllers/    Auth, Admin, Entrainements, Evenements, Reunions
├── models/         User_model, Entrainement_model, Evenement_model,
│                   Reunion_model, Participation_model
└── views/          auth/, admin/, entrainements/, evenements/,
                    reunions/, dashboard/ (selon rôle)
Pages et rôles :
  • Invité : présentation du club
  • Joueur : liste des joueurs et des évènements
  • Entraîneur : gestion des présences et du travail réalisé
  • Membre de l'organisation : CRUD complet sur entraîneurs (avec gestion des remplacements) et évènements
Authentification par email (généré par déclencheur) ou pseudo, avec redirection automatique selon statut.
Période de réalisation Année 2024–2025 (13 séances S1→S13, BTS SIO 2ème année). Évaluation finale après les vacances. Stage en parallèle du 13/01/2025 au 14/02/2025.
Outils & technologies SGBD : SQL Server 2019, SQL Server Management Studio (SSMS), SSMA (migration MySQL → SQL Server), drivers ODBC
Modélisation : Looping (MCD avec héritage et réflexivité)
Backend : Transact-SQL (Procédures stockées, Déclencheurs AFTER/INSTEAD OF, Vues, fonctions)
Sécurité : hachage HASHBYTES SHA2_512, Try/Catch, JWT
Framework : CodeIgniter (MVC, ORM, routage, vues, assets, API REST)
Serveur : Laragon, Apache
IDE : Visual Studio Code, Composer
Concepts d'analyse abordés Héritage et contraintes (partition, exclusion, totalité), id relatif, réflexivité (associations unaires fonctionnelles et non fonctionnelles), formes normales, intégrité référentielle
Lieu Lycée CCI de Nîmes
Cadre de réalisation Atelier de Professionnalisation 3 (AP3) — BTS SIO 2ème année option SLAM
Mode de réalisation Travail individuel avec contrôles intermédiaires (Cours 1 : MVC/MCD/SQL ; Cours 2 : TR/PS/ORM) et 2 évaluations AP3 (BDD puis CodeIgniter)
